# 量化交易系统升级总结

**升级日期**: 2026-03-06  
**版本**: v2.0 (盘中实时监控)

---

## 🎯 升级背景

### 原系统问题

老板您指出的两个核心问题：

1. **❌ 缺乏 K 线实时跟踪**
   - 只在收盘后获取 K 线
   - 无法识别盘中突破
   - 错过回调买点

2. **❌ 交易时间滞后 (更严重)**
   ```
   原流程:
   15:00  收盘
   15:30  分析
   15:45  信号
   次日 09:30  买入 → 成本高/买不进
   ```

   **T+1 制度下的致命问题**:
   - 龙头股可能一字板，买不进
   - 好的买点 (盘中回调) 已错过
   - 次日买入成本更高

---

## ✅ 升级方案

### 新系统：盘中实时监控

```
新流程:
09:25  集合竞价 → 识别强势股
09:30  开盘 → 监控资金流
10:00  扫描 K 线突破 → 发现买点
10:30  监控回调到位 → 龙头回调策略
14:00  发现尾盘机会 → 次日溢价
14:50  收盘前决策 → 确定买入
15:00  收盘 → 当日成交，次日可卖
```

**核心优势**:
- ✅ 捕捉盘中最佳买点
- ✅ 避免收盘后滞后
- ✅ 适合 T+1 交易制度
- ✅ 成本最优

## 📊 新增功能

### 1. 实时 K 线获取

**文件**: `src/realtime_monitor.py`

**功能**:
- ✅ 支持多种周期 (1/5/15/30/60 分钟，日线，周线)
- ✅ 实时行情获取 (每秒更新)
- ✅ 批量获取支持
- ✅ 自动重试机制

**示例**:
```python
from realtime_monitor import RealtimeKLineFetcher

fetcher = RealtimeKLineFetcher()

# 获取日线
kline = fetcher.get_realtime_kline('000533', count=60)

# 获取 5 分钟 K 线
kline_5m = fetcher.get_realtime_kline('000533', count=60, scale='5')

# 获取实时行情
quote = fetcher.get_realtime_quote('000533')
print(f"当前价：{quote['price']}元")
```

### 2. 盘中三大策略

**文件**: `src/intraday_strategy.py`

#### (1) 早盘突破 (09:30-10:30)

**条件**:
- ✅ 高开 2-5%
- ✅ 开盘后继续上涨
- ✅ 成交量放大 (量比>1.5)

**参数**:
- 止损：-5%
- 止盈：+15%
- 仓位：10%

**逻辑**:
```python
# 高开幅度
gap_up = (open_price - prev_close) / prev_close * 100

# 当前涨幅
change_pct = (current_price - prev_close) / prev_close * 100

# 量比
volume_ratio = current_vol / avg_vol

if 2 <= gap_up <= 5 and change_pct > gap_up and volume_ratio > 1.5:
    buy()
```

---

#### (2) 盘中回调 (10:30-14:30)

**条件**:
- ✅ 从日内高点回调 3-5%
- ✅ 缩量 (量比<0.8)
- ✅ 企稳 (不创新低)

**参数**:
- 止损：-5%
- 止盈：+10%
- 仓位：10%

**逻辑**:
```python
# 回调幅度
pullback = (intraday_high - current_price) / intraday_high * 100

# 量比
volume_ratio = recent_vol / avg_vol

# 企稳检查
if 3 <= pullback <= 5 and volume_ratio < 0.8 and not making_new_low:
    buy()
```

---

#### (3) 尾盘买入 (14:30-15:00)

**条件**:
- ✅ 全天强势 (涨幅 3-7%)
- ✅ 尾盘不跳水
- ✅ 在均价上方

**参数**:
- 止损：-5%
- 止盈：+10% (次日溢价)
- 仓位：10%

**逻辑**:
```python
# 涨幅
change_pct = (current_price - prev_close) / prev_close * 100

# 尾盘稳定性
if 3 <= change_pct <= 7 and current_price > avg_price * 0.98:
    buy()
```

## ⚠️ 注意事项

### 1. 交易时间运行

**必须在交易时间运行**:
- 上午：09:30-11:30
- 下午：13:00-15:00

**休市时间无实时数据**。

---

### 2. 网络要求

- ✅ 稳定的网络连接
- ✅ 建议有线网络
- ⚠️ 避免 WiFi 波动

---

### 3. 监控股票数量

**建议**:
- 早盘：监控 5-10 只龙头
- 盘中：监控 10-20 只
- 尾盘：监控 20-30 只

**过多会增加延迟**。

---

### 4. 信号过滤

**建议设置信心阈值**:
- 早盘突破：信心≥0.7
- 盘中回调：信心≥0.6
- 尾盘买入：信心≥0.75
